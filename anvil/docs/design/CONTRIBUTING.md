# Contributing / PR rules

## Branch

- Work from feature branches  
- One milestone task group per PR when possible  

## PR checklist

- [ ] Tasks in `20` marked for completed items  
- [ ] `pnpm test` / `anvil validate` / `anvil test` green for touched examples  
- [ ] No Phaser imports outside render-phaser  
- [ ] Specs updated if public API changed  
- [ ] Observe schema changes = major version note  

## Commits

- Imperative subject; optional `M1:` prefix for milestone  

## Review

- ACI breaks require design review  
- Genre rule changes require S-* update in same PR
